# 產品需求文件 (PRD) - 台北捷運快速轉乘與出站導航系統 (Taipei MRT Rapid Transfer & Exit Navigation System)

本專案為資訊管理系統課程之期末成果發表專案，旨在開發一個結合「精準轉乘」與「智慧出站」的輕量化捷運優化工具。

---

## 一、 核心價值與痛點 (Value & Pain Points)

### 1. 目標用戶 (Target Users)
* **趕早八的校園學生**：時間極度緊迫，需要精準計算出每一班車的銜接時間，並從最靠近接駁點或校門的出口出站。
* **學校外籍教師 / 國際交換生**：對台北捷運路網、複雜車站結構（如台北車站、忠孝復興站）與周邊地理位置不熟悉，容易走錯出口。

### 2. 現有痛點 (Pain Points)
* **轉乘動線不直覺**：官方 App 缺乏「下車後走哪一個車廂最快到轉乘電扶梯」的微觀指引，導致使用者在月台盲目尋找動線。
* **出口選錯繞遠路**：大型捷運站（如公館站、大安站、中山站）出口眾多。一旦走錯出口，往往需要多花 5-10 分鐘過馬路或繞行，對於趕時間的學生或迷路的外師是極大痛點。
* **出口設施資訊不透明**：攜帶行李、外師推嬰兒車或受傷提重物的學生，常走到出口才發現該出口「只有長樓梯、沒有電扶梯/電梯」，造成極大不便。

### 3. 產品定位 (Product Positioning)
本系統定位為**「捷運微觀導航：從月台到出口的一條龍優化工具」**。整合轉乘車廂資訊與各出口的周邊地標、無障礙設施，用最直覺的介面解決使用者的時間焦慮與空間迷航。

## 二、 核心功能範疇 (MVP Scope)

本系統以實用、結構清晰、可於發表會上完整展示的 MVP（最小可行性產品）為範圍：

### 1. 【核心功能】快速轉乘路徑推薦與車廂導航
* **需求描述**：使用者輸入起點站與終點站後，系統不僅規劃乘車路線，更直接指出**「建議在起點站搭乘第幾車廂」**，實現下車即轉乘。
* **驗收標準 (AC)**：
    * 使用者選擇「板橋站」到「科技大樓站」，系統應計算出需在「忠孝復興站」轉乘。
    * 系統畫面需明確提示：「在板橋站上車時，請在 **第 3 車廂** 就位，下車後即為往文湖線的電扶梯。」

### 2. 【核心功能】智慧出站與出口資訊查詢
* **需求描述**：使用者可直接選定或搜尋特定車站，系統需列出該站**所有出口的周邊地標、校園方向**，並清晰標示**該出口的硬體設施（樓梯/電扶梯/電梯）**。
* **驗收標準 (AC)**：
    * 使用者查詢「公館站」，系統應明確顯示：
        * **3 號出口**：往台灣大學校門（硬體：有雙向電扶梯）
        * **1 號出口**：往水源市場（硬體：僅有樓梯）
    * 支援中英文關鍵字（如：NTU / 台大），方便外師快速對應正確出口。

### 3. 【次要功能】車站關鍵設施顯示
* **需求描述**：提供車站基本設施簡圖或清單，快速標示出詢問處、洗手間（付費區內/外）位置。
* **驗收標準 (AC)**：
    * 使用者在查看車站資訊時，能一目了然該站的洗手間是在「站內」或「站外」。

### 4. 【超出範疇 (Out of Scope)】本版本暫不實作
* **即時列車動態連線**：暫不串接政府 API 取得列車即時倒數。
* **會員註冊與歷史紀錄**：不進行用戶資料庫管理，維持免登入即可使用的輕量化體驗。

## 三、 使用者流程與邏輯 (User Flow & Logic)

### 1. 主流程路徑 (Happy Path)
[進入系統首頁]
│
├─► 【功能 A：轉乘查詢】 ──► 輸入起訖站 ──► 顯示：推薦路線 + 「第 X 車廂第」
│
└─► 【功能 B：出站查詢】 ──► 選擇/輸入目標站 ──► 顯示：各出口列表、周邊地標與設施(電梯/樓梯)

### 2. 關鍵業務與技術邏輯
* **資料庫關聯設計**：
    * 系統內部資料包含兩大核心資料表：`Station_Transfer_Table`（儲存轉乘車廂對照）與 `Station_Exit_Table`（儲存各站出口的地標與設施資訊）。
    * 透過「車站代碼 (Station ID)」將轉乘資訊與出口資訊進行關聯。
* **例外狀況防呆**：
    * 若搜尋出口時找不到對應地標，系統應友善提示：「查無此地標，請嘗試輸入車站名稱」。

## 四、 成功指標與未來規劃 (KPIs & Future Roadmap)

### 1. 發表會衡量指標 (Success Metrics for Presentation)
* **資訊檢索效率**：使用者在 2 個點擊內（選擇車站 -> 查看出口），即可得知目標出口是否有電扶梯，資訊獲取時間縮短至 3 秒內。
* **展示流暢度**：資料採輕量化靜態資料庫（如 JSON 格式）驅動，前端載入與切換無延遲。
* **高實用價值**：成功展示「早八趕車情境」，證明系統能實質幫學生與外師省下盲目找路的時間。

### 2. 後續優化規劃 (Future Roadmap)
* **Phase 2 - 出口導航一體化**：未來將結合轉乘與出站功能。使用者輸入起訖點時，可直接勾選「我的目的地是台大校門」，系統便會自動規劃「起點車廂 ──► 轉乘車廂 ──► 建議出站出口」的一條龍最佳路徑。
